## 💡 Key Concepts Explained Simply

### MVVM Architecture
- **Model**: GameBoard (game logic)
- **ViewModel**: GameViewModel (state management)
- **View**: GameActivity + CandyView (UI display)
- **Why**: Separation of concerns, easier to test

### LiveData
- Reactive data container
- Notifies UI when data changes
- UI updates automatically
- No manual observer management needed

### Coroutines
- Async operations without threads
- Light-weight and efficient
- Used for animation timing delays
- Keeps UI responsive

### Custom View (CandyView)
- Canvas-based drawing (more performant than views)
- Renders 8×8 grid of colored circles
- Handles touch events
- Updates when board data changes
